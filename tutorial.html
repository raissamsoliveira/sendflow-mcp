Sobre este tutorial

Este guia usa Node.js — uma alternativa mais simples ao servidor em Python. Não é necessário criar ambientes virtuais, instalar Python ou configurar dependências complexas. Tudo funciona com um único npm install.

02
01 · Conceito

O que é o MCP?

O Model Context Protocol (MCP) é um padrão aberto criado pela Anthropic que permite ao Claude se conectar a ferramentas e serviços externos de forma segura e padronizada. Pense nele como uma "tomada universal" entre o Claude e qualquer API ou sistema que você quiser integrar.

Sem o MCP, o Claude não consegue chamar APIs externas diretamente. Com o MCP, você cria um servidor local que expõe as capacidades de uma API como ferramentas — e o Claude aprende a usá-las automaticamente a partir de linguagem natural.

Em outras palavras:

Você diz "liste minhas campanhas", o Claude entende que deve chamar a ferramenta list_campaigns, o servidor MCP monta a requisição HTTP autenticada e devolve o resultado — tudo sem você escrever uma linha de código na hora.

Como funciona na prática

  • 1
    Você cria um servidor MCP local

    Um arquivo Node.js que declara ferramentas e define como cada uma chama a API do Sendflow.

  • 2
    O Claude Code carrega o servidor

    Ao iniciar, o Claude Code descobre as 41 ferramentas disponíveis e aprende seus parâmetros automaticamente.

  • 3
    Você conversa em linguagem natural

    O Claude escolhe a ferramenta certa, monta a chamada, executa e devolve a resposta formatada para você.

Por que não usar a API diretamente?

Você pode — mas com o MCP você não precisa lembrar endpoints, parâmetros ou formatos de data. O Claude cuida disso. Você foca no que quer fazer, não em como fazer.

05 · Projeto

Estrutura do projeto

O servidor MCP é composto por apenas 5 arquivos. Estrutura simples, sem configuração complexa.

Estrutura da pasta
sendapi/
├── index.js         # servidor MCP — 41 ferramentas (~500 linhas)
├── package.json     # dependências do projeto
├── .env.example     # template da API key (nunca commite o .env real)
├── .gitignore       # ignora node_modules/ e .env
├── SETUP.md         # este guia de instalação
└── node_modules/    # criado pelo npm install (não editar)
package.json
{
  "name": "sendflow-mcp",
  "version": "2.0.0",
  "type": "module",            // ESM — obrigatório
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  },
  "dependencies": {
    "@modelcontextprotocol/sdk": "^1.9.0"
  }
}
Por que v1.9.0 e nao a mais recente?

A partir da v1.10.x, o SDK passou a usar zod v4, que tem incompatibilidades de importacao com Node.js v22 em ambientes ESM. A v1.9.0 e estavel e cobre todos os recursos necessarios.

.env.example
# Copie para .env e preencha com sua chave real
SENDFLOW_API_KEY=send_api-coloque_sua_chave_aqui

06 · Servidor

O servidor MCP (index.js)

O index.js é o coração da integração. Ele usa o SDK do MCP para expor as ferramentas da SendAPI ao Claude Code. O arquivo tem ~500 linhas e está pronto para usar — não é necessário editá-lo.

Estrutura do index.js — visão geral
// 1. Importações do SDK MCP
import { Server } from "@modelcontextprotocol/sdk/server/index.js"
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js"

// 2. Configuração — lê a API key do ambiente
const API_KEY = process.env.SENDFLOW_API_KEY
const BASE_URL = "https://sendflow.pro/sendapi"

// 3. Helper HTTP — faz as chamadas autenticadas
async function callAPI(method, path, body) { ... }

// 4. Definição das 41 ferramentas (TOOLS array)
const TOOLS = [
  // Campanhas (10), Grupos (4), Acoes (9), Mensagens diretas (4),
  // Templates (4), Contas (7), Bloqueios (2), Verificacao (1)
]

// 5. Handler — executa a ferramenta certa quando o Claude chama
async function handleTool(name, args) {
  switch (name) { ... 41 cases ... }
}

// 6. Inicialização do servidor via stdio
const server = new Server({ name: "sendflow-mcp", version: "2.0.0" }, ...)
await server.connect(new StdioServerTransport())
Cada ferramenta tem 3 partes:

name — identificador único (ex: list_campaigns) · description — o Claude usa isso para saber quando chamar a ferramenta · inputSchema — os parametros esperados (o Claude monta automaticamente)

O servidor roda via stdio

O Claude Code inicia o processo node index.js automaticamente e se comunica via stdin/stdout (stdio). Voce nao precisa deixar nenhum servidor rodando em background — o Claude Code gerencia o ciclo de vida do processo.

07 · Configuracao

Instalar e registrar

Com os arquivos criados, sao apenas 3 comandos para colocar tudo no ar.

  • 1
    Instalar as dependencias

    Abra o terminal na pasta sendapi e execute:

cd "C:\Users\raiss\Desktop\sendapi"
npm install
  • 2
    Registrar o servidor no Claude Code

    O flag --scope user torna o servidor disponivel em qualquer pasta do seu computador.

claude mcp add sendflow --scope user \
  --env SENDFLOW_API_KEY=send_api-SUA_CHAVE_AQUI \
  -- node "C:\Users\raiss\Desktop\sendapi\index.js"
  • 3
    Confirmar que foi registrado

    Deve aparecer sendflow na lista:

claude mcp list
# sendflow   node C:\Users\raiss\Desktop\sendapi\index.js
Precisa substituir o servidor?

Se precisar atualizar o caminho ou a API key:
claude mcp remove sendflow — e depois adicione novamente com o comando do passo 2.

Alternativa: Claude Desktop

Se preferir usar o Claude Desktop, adicione o bloco abaixo ao arquivo %APPDATA%\Claude\claude_desktop_config.json:
{ "mcpServers": { "sendflow": { "command": "node", "args": ["C:\\...\\index.js"], "env": { "SENDFLOW_API_KEY": "..." } } } }

09 · Uso

Uso no dia a dia

A partir daqui e tudo linguagem natural. O Claude escolhe a ferramenta certa das 41 disponiveis e cuida dos detalhes tecnicos. Exemplos do que voce pode fazer:

"

Me mostra o analytics das minhas ultimas 3 campanhas e qual teve mais cliques hoje.

→ get_campaign_analytics
"

Agende uma mensagem 'Promocao relampago!' para a campanha X amanha as 10h com velocidade normal.

→ send_text_to_campaign
"

Verifique se os numeros 11999999999 e 11888888888 estao bloqueados para a campanha Y.

→ verify_number
"

Crie uma campanha 'Black Friday 2026' e adicione o grupo 120363292004848696@g.us nela.

→ create_campaign + add_release_group
"

Bloqueie o numero 5511999999999 como 'Spammer' e confirme o bloqueio.

→ block_number + list_blocked_numbers
"

Me mostra o QR code da conta ACC123 — preciso reconectar o WhatsApp.

→ connect_account + get_account_qrcode
Dica: combine com arquivos

O Claude Code pode combinar as ferramentas do Sendflow com leitura e escrita de arquivos. Por exemplo: "Leia a planilha contatos.xlsx, verifique quais numeros estao bloqueados na campanha X e me gere um relatorio." O Claude orquestra tudo automaticamente.

10 · Solucao de problemas

Troubleshooting

SENDFLOW_API_KEY nao definida
O servidor subiu mas nao encontrou a API key.
Confirme que o --env SENDFLOW_API_KEY=... foi passado no claude mcp add. A key no parametro --env tem precedencia sobre o arquivo .env.
401 Unauthorized
API Key invalida, expirada ou sem permissao.
Gere uma nova chave no painel do Sendflow. Remova o servidor (claude mcp remove sendflow) e registre novamente com a nova chave.
MCP server sendflow already exists
O servidor ja estava registrado de uma sessao anterior.
Apenas prossiga — a mensagem e informativa, nao e um erro. Ou remova e adicione novamente: claude mcp remove sendflow.
claude nao e reconhecido / nao e um comando
O PATH do npm global nao esta no ambiente.
Execute npm config get prefix para descobrir onde o npm instalou. Adicione esse caminho ao PATH nas variaveis de ambiente do Windows.
Claude Code on Windows requires git-bash
Git for Windows nao instalado ou variavel nao definida.
Instale o Git em git-scm.com e defina a variavel de ambiente CLAUDE_CODE_GIT_BASH_PATH apontando para C:\Program Files\Git\bin\bash.exe.
ERR_MODULE_NOT_FOUND ou zod/v4 import error
Versao incompativel do SDK com Node.js v22.
Garanta que o package.json usa "@modelcontextprotocol/sdk": "^1.9.0" e rode npm install novamente.
